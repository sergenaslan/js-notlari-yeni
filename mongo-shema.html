mongoose.Schema() Fonksiyonu

mongoose.Schema() fonksiyonu, bir collection içerisine kaydedilecek documentlerin field alanlarında tutulacak verilerin veri tiplerini, varsayılan değerlerini, zorunlu olup olmadığını ve çeşitli doğrulama (validation) kurallarını tanımlamamızı sağlayan bir document yapısı (şeması) oluşturmamızı sağlar. mongoose.Schema({}) gerçek projelerde sadece alan ve tip tanımlamak için değil, veri güvenliği, doğrulama (validation), performans ve iş kuralları koymak için kullanılır. 

DATABASE → kapinda_kuruyemis
COLLECTION → products
DOCUMENT → { name: "İncir", price: 120 }
FIELD → name / price

 DATABASE                 DATABASE
 └── COLLECTION            └── TABLE
      └── DOCUMENT              └── ROW
           └── FIELD                └── COLUMN

Schema Definition

Bir document içindeki TÜM field’ların yapısını tanımlama işlemidir.

Schema = ev planı
Field = odanın iç tasarımı

// Burada yapılan işlem Schema Definition
const productSchema = mongoose.Schema({
  name: String,
  price: Number,
  stock: Number
})
// Veri Tipleri → String, Number, Boolean, Date, Array, ObjectId, Mixed, Buffer

Field Configuration

Tek bir field’ın özelliklerini belirleme işlemidir.

const productSchema = new mongoose.Schema(
{
name: {
  type: String,
  required: true,   → zorunlu alan Bu alan olmadan kayıt yapılamaz
  trim: true,       → boşlukları siler "   incir   "
  minlength: 5,     → ürün adı 5 karakterden küçük olamaz
  maxlength: 30,    → ürün adı 30 karakterden büyük olamaz
  alias: "urunAdi"  → product.urunAdi şeklinde erişim sağlayabilmek için
},
slug: {
  type: String,
  lowercase: true,    → küçük harfe çevirir
  unique: true,      → organik-incir //Slug genelde SEO için kullanılır
  index: true,        → performans artırır
},
price: {
  type: Number,
  min: 0             → ürün fiyatı 0 az olamaz
  max: 999          → ürün fiyatı 999 fazla olamaz
  validate: {        → Custom validation yazılabilir:
    validator: function(v) {
      return v > 0;
    },
    message: "Fiyat 0'dan büyük olmalı"
  },
  set: v => Math.round(v)  → Veri kaydedilirken değiştirilebilir
},
tags: [   →tags alanı bir liste (array) olacak, ve listenin içindeki her eleman String olacak
  {
    type: String  →örnek => "tags": ["organik", "yerli", "dogal"]
  }
],
stock: {
  type: Number,
  default: 0         → Varsayılan değer
},
category: {
  type: String,
  enum: ["kuruyemis", "paket"]  → sadece belirli değerler kabul edilir
},
createdBy: {
  type: mongoose.Schema.Types.ObjectId, → User tablosundan gelecek
  ref: "User"
},
updatedAt: {
  timestamps: true      → Mongo otomatik ekler:
},
email: {
  type: String,        →  match (regex doğrulama)
  match: /.+\@.+\..+/
},
address: {
 city: String,
 district: String
},
password: {
 type: String,      → Varsayılan olarak alanı gizler
 select: false
},
createdAt: {
 type: Date,      → Alan değiştirilemez olur
 immutable: true,
 timestamps: true
}
});

schema.methods

schema.methods, document seviyesinde çalışan özel fonksiyonlar ekleyebilmemizi sağlar

//Shema tanımlama
const productSchema = new mongoose.Schema({
  name: String,
  stock: Number
});

// Method ekleme
productSchema.methods.isInStock = function() {
  return this.stock > 0; //this o anki document’un stock değeri
};

// model
const Product = mongoose.model("Product", productSchema);

📘 Kullanım örneği
//Database’ten ürün çekelim:
const product = await Product.findOne();

//Şimdi method kullanılır:
product.isInStock();  //true-false

Middleware

belirli bir işlemden önce veya sonra otomatik çalışan fonksiyondur

save, validate, remove, deleteOne, updateOne, find, findOne, findOneAndUpdate 
yukarıdaki işlemler esnasında eklenebilinir.
1️⃣ pre Middleware
schema.pre("save", function() {}) // İşlemden önce çalışır
2️⃣ post middleware 
schema.post("save", function() { console.log(log yazmak) }) // İşlemden sonra çalışır
log yazmak, mail göndermek, notification göndermek, audit sistemi kurmak, cache temizlemek

// save işleminden önce çalışır  document nasıl davranacak?
productSchema.pre("save", function(next) {
 this.slug = this.name
  .toLowerCase()
  .replaceAll(" ", "-");
 next();
});

userSchema.pre("save", function(next) {
 this.password = hash(this.password);
 next();
});

productSchema.post("save", function(doc) {
 console.log("Ürün kaydedildi:", doc.name); 
});

orderSchema.post("save", function(order) {
 sendSMS(order.phone);
});

schema.statics

schema.statics, mongoose modeline özel fonksiyon eklemek için kullanılır. tek bir kayıt üzerinde değil, koleksiyonun tamamı üzerinde çalışan fonksiyonlardır.

// schema.statics, mongoose modeline özel fonksiyon eklemek için kullanılır.

//Schema:
const productSchema = new mongoose.Schema({
 name: String,
 stock: Number,
 isActive: Boolean
});

//statics ekleyelim:
productSchema.statics.findAvailableProducts = function() {

 return this.find({    //this burada modeli temsil eder.
  stock: { $gt: 0 },
  isActive: true
 });
};
//model
const Product = mongoose.model("Product", productSchema);

//kullanım
const products = await Product.findAvailableProducts(); //stokta olan ve aktif ürünler gelir.

tüm aktif siparişler getirme
const orderSchema = new mongoose.Schema({
 orderNumber: String,
 orderStatus: String,
 totalAmount: Number
});

orderSchema.statics.findActiveOrders = function() {
 return this.find({
  orderStatus: "active"
 });
};

const Order = mongoose.model("Order", orderSchema);
const activeOrders = await Order.findActiveOrders();  tüm aktif siparişler gelir

productSchema.plugin()

plugin, mongoose schema’ya tekrar kullanılabilir özellikler eklemek için kullanılan genişletme yapısıdır.

plugin, schema’ya hazır özellikler ekleyen tekrar kullanılabilir modüllerdir.

function slugPlugin(schema) {

 schema.pre("save", function(next) {

  this.slug = this.name
   .toLowerCase()
   .replaceAll(" ", "-");

  next();

 });

}

productSchema.plugin(slugPlugin);  otomatik slug üretir
